- [ ] Check endpoints used for both lair and agents such as /stream, /branches, etc and see why they are necessary
- [ ] Audit how lair-side secrets (API keys, Noise private key) are stored on the host. Today `octo init` writes them as plaintext lines in `~/.octo/lair-env` (mode 0600) and they appear in `docker inspect lair` under `Config.Env`. Consider switching to Docker secrets / a separate mount that doesn't show up in `inspect`, or at minimum confirm `inspect` access is gated behind host root.
- [ ] Client-key allowlist + first-connection ack UI — `noise_handshake` already captures the client static key from snow's `get_remote_static()` and logs it, but it's never checked. Persist a `known_clients.json` on lair, gate new client pubkeys behind a first-connection ack flow in the mobile UI (approve / reject), and reject handshakes from unknown keys after the first run. Replaces today's QR-only TOFU model where anyone with the QR can connect indefinitely.
- [ ] Children generate their own Noise keypair on first boot — today `lair::exec_create_agent` injects the parent's hex-encoded keypair into each child via the `NOISE_PRIVATE_KEY` env var, so leaking one child's container env compromises lair and every sibling. Have child servers run `load_or_generate_keypair` against their own `/data/noise_key.bin`, register their pubkey back to lair via a small HTTP endpoint, and let lair store the per-child pubkey in the agent registry instead of broadcasting its own.
- [ ] Front the push-notification relay (`octorelay.directto.link`) with Cloudflare to absorb DDoS / bad-signature floods before they reach the t4g.nano's CPU credits. Today the box is bare on a public Elastic IP; if `/notify` ever attracts spammers, Ed25519 verifies are cheap but can still chew through burst credits at high enough volume. Cloudflare in front gives free L7 caching, rate-limits, bot challenges, and a single knob to drop traffic without SSH'ing the instance.
